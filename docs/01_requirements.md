# 要件定義書 - Survey Aggregator

## 基本情報
- **アプリ名**: Survey Aggregator（アンケート集計ツール）
- **対象プラットフォーム**: Web（ブラウザ、Chrome想定）
- **初回リリース目標**: 未定（MVP完成次第、友人に共有）
- **主な利用者**: 社長の友人（非エンジニア）
- **作成者**: 社長（app-dev部門による開発）

## 背景・目的
- 友人がイベント等でGoogleフォームのアンケートを取得し、CSVエクスポートした結果を集計したい
- 単純集計（各設問の回答数・割合）を手動で行うのは手間がかかる
- 会社データのため、外部AIサービスや外部サーバーに一切送信しない構成にしたい
- 友人が技術者ではないため、ブラウザで開いてドラッグ&ドロップで完結する操作性が必要

## スコープ

### 今回やること（MVP）
- Googleフォームの回答CSVファイルの読み込み
- 全設問の単純集計（度数・割合）をExcel出力
- 複数選択回答の分解集計（カンマ+スペース区切り）
- 自由記述の一覧出力
- 列タイプの自動判定＋手動上書き機能
- 集計対象列のON/OFF選択
- クロス集計機能（画面上で条件を選んで集計）
- クロス集計結果のExcel出力
- GitHub Pages等での静的ホスティング公開

### 今回やらないこと（将来対応）
- 認証機能（Google認証・パスワード等）
- データベース保存（集計結果の履歴保存等）
- グラフ描画（棒グラフ・円グラフ等の画像生成）
- OR条件・不等号・範囲フィルタ（ANDと等号のみ対応）
- 数値列の自動ビン分割（例: 年齢23→20代）
- Googleフォーム以外のCSV形式への対応（当面は標準形式のみ想定）
- 集計設定のプリセット保存
- 複数人での同時利用・共有機能
- ChatGPT等のAIサービス連携
- モバイル端末対応（PCブラウザのみ）

## 集計ロジック仕様

### 単純集計
- **単一選択列**: 選択肢ごとに「回答数・回答者全体に対する割合」を算出
  - 分母: 有効回答者数（空欄除外）
- **複数選択列**: カンマ+スペース（`, `）で分解し、選択肢ごとに「回答数・回答総数に対する割合」を算出
  - 分母: **回答総数ベース（分解後の選択件数の合計値）**
  - 定義の明確化: 例えば100人の回答者のうち、各人が平均1.85個の選択肢を選んだ場合、回答総数=185件となる。各選択肢の割合は「その選択肢の件数 ÷ 185」で算出する
  - この方式では全選択肢の割合合計が100%となる（回答者数ベースではない点に注意）
- **数値列**: 回答数、平均値、中央値、最大値、最小値を表示
- **自由記述**: 集計せず、別シートに回答を一覧表示
- **集計対象外**: 何も出力しない

### クロス集計
- **条件指定方式**: 設問ドロップダウン → 選択肢ドロップダウン（連動）
- **条件結合**: AND のみ
- **比較演算**: 等号（=）のみ
- **複数選択列のフィルタ**: 「その選択肢を含む回答者」として抽出（選択肢を1つでも選んでいれば対象）
- **集計対象設問**: 単純集計と同じロジック（単一選択・複数選択・数値・自由記述）
- **分母の扱い**:
  - 単一選択の集計対象: フィルタ後の有効回答者数
  - 複数選択の集計対象: フィルタ後の回答総数ベース（分解後の選択件数の合計）
- **該当者0人の場合**: 画面上で「該当者なし」と明示。Excel出力時も同様に表示
- **条件が1件も設定されていない状態**: 「条件を1つ以上追加してください」と案内メッセージを表示し、集計を実行しない（単純集計と重複するため）

### 列タイプ自動判定ロジック
優先順位（上から順に評価し、最初にマッチしたタイプを採用）:
1. **タイムスタンプ列**: 列名に「タイムスタンプ」「timestamp」を含む、または値が日時形式 → 集計対象外（自動でOFF）
2. **メールアドレス列**: 列名に「メール」「email」「mail」を含む、または値がメアド形式 → 集計対象外（自動でOFF）
3. **複数選択**: セル値に `, ` (カンマ+スペース) を含む行が **全有効行の20%以上** ある → 複数選択
4. **単一選択**: ユニーク値が **5個以下** かつ自由記述でない → 単一選択
5. **数値**: 全ての非空セルが数値として解釈できる → 数値
6. **自由記述**: 上記のいずれにも該当しない → 自由記述

- 判定結果はUIで確認・変更可能
- 閾値（20%、5個）は実装時に定数として定義し、調整可能にする
- **数値より複数選択を先に判定する**のは、選択肢が数字のみの複数選択列（例: 「1, 2, 3」）を数値列と誤判定するリスクを避けるため

## 非機能要件

### パフォーマンス
- 回答件数 **数百〜数千件** で動作保証
- 1万件程度までは実用速度で動作（5秒以内の集計完了を目安）
- それを超える件数は保証外（ブラウザメモリ制約）

### 対応ブラウザ/端末
- **Chrome最新版**（デスクトップ）を主な動作保証対象
- Safari・Edge・Firefoxは動作を想定するが保証外
- スマホ・タブレット非対応

### セキュリティ
- **CSVデータは一切サーバーに送信しない**（ブラウザ内のJavaScriptで完結）
- **データベース・認証サービスを使用しない**
- **AIサービスへのデータ送信を行わない**
- ホスティングサーバー（GitHub Pages等）には静的ファイルのみ配置
- アクセス制御なし（URLを知っている人は誰でも利用可能）
  - 理由: ツール側にデータが無いため、URL流出のリスクは実質ゼロ
  - ツールを使うことで得られるのは「集計機能」のみで、他者のデータは見られない

### データ量の想定
- 想定CSVサイズ: 数百KB〜数MB（数百〜数千件）
- 設問数: 10〜50問程度を想定
- ユーザー数: 社長の友人1名から開始、徐々に増える可能性あり

### 運用
- 社長（または指示を受けたエンジニア）がコード修正・再デプロイを担当
- 友人側の操作は「URLを開く→CSV投入→Excel保存」のみ
- アップデート時は友人側の操作不要（URL再アクセスで最新版）

## 利用シナリオ

### シナリオ1: イベントアンケートの基本集計
1. 友人がイベント後、Googleフォームから回答CSVをダウンロード
2. ツールのURLをブラウザで開く
3. CSVファイルをドラッグ&ドロップ
4. 列タイプ自動判定結果を確認、必要があれば手動調整
5. タイムスタンプ・メアド等を集計対象から除外
6. 「Excel出力」ボタンで単純集計のExcelをダウンロード

### シナリオ2: クロス集計で属性別の傾向を見る
1. シナリオ1の状態から、「クロス集計」タブに切り替え
2. 「＋条件を追加」→ 設問「年代」→ 選択肢「30代」を選択
3. 「＋条件を追加」→ 設問「性別」→ 選択肢「女性」を選択（AND結合）
4. 「集計対象設問」→「きっかけ」を選択
5. 結果が画面に表示される（該当者数、各選択肢の度数・割合）
6. 「Excel出力」で結果をダウンロード
