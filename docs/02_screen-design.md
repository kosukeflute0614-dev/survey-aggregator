# 画面設計書 - Survey Aggregator

## 画面一覧

| # | 画面名 | URL（想定） | 認証 | 概要 |
|---|--------|------------|------|------|
| 1 | トップ画面（CSV投入） | `/` | なし | CSVファイルのドラッグ&ドロップ受付 |
| 2 | 列設定画面 | `/`（状態遷移） | なし | 自動判定結果の確認・列タイプ変更・集計対象ON/OFF |
| 3 | 集計画面 | `/`（状態遷移） | なし | 単純集計のプレビューとExcel出力、クロス集計UI |

※ シングルページアプリケーション（SPA）構成。URL遷移は発生せず、画面状態で切り替える。
※ 実際のベースURLはGitHub Pagesのサブパス構成になる（例: `https://<username>.github.io/survey-aggregator/`）。絶対パスは `/survey-aggregator/` 起点になるため、Viteの `base` 設定で対応する。

## 画面遷移図

```
[1. CSV投入画面]
    │
    │ CSVドロップ
    ▼
[2. 列設定画面]
    │
    │ 「集計画面へ進む」ボタン
    ▼
[3. 集計画面]
    ├─ タブ「単純集計」
    │   └─ Excel出力ボタン → .xlsxダウンロード
    │
    └─ タブ「クロス集計」
        ├─ 条件追加UI
        ├─ 結果表示エリア
        └─ Excel出力ボタン → .xlsxダウンロード

※ 各画面から前の画面に戻るリンクあり
※ 「CSVを差し替える」ボタンで画面1に戻れる
```

## 各画面の詳細

### 画面1: CSV投入画面

- **URL**: `/`（初期状態）
- **アクセス権限**: 誰でも
- **主な要素**:
  - **ヘッダー**: アプリ名「Survey Aggregator」、簡単な説明（例: 「CSVをドロップすると集計できます」）
  - **メインコンテンツ**:
    - 大きなドロップゾーン（「CSVファイルをここにドラッグ&ドロップ」）
    - または「ファイルを選択」ボタン（クリックでファイルダイアログ）
    - 対応形式の注記（例: 「Googleフォームからエクスポートした.csvファイル（UTF-8）」）
  - **フッター**:
    - データ取扱いに関する注記（「このツールはデータをサーバー送信しません。全てブラウザ内で処理されます」）
- **操作と遷移**:
  - CSVをドロップまたは選択 → パース処理 → 成功したら画面2へ
  - CSVパースエラー時: エラーメッセージ表示（「ファイル形式が正しくありません」等）

### 画面2: 列設定画面

- **URL**: `/`（状態遷移）
- **アクセス権限**: 誰でも
- **主な要素**:
  - **ヘッダー**: 「列設定」、「← CSVを差し替える」リンク
  - **サマリー情報**: 読み込んだ列数、回答数（例: 「18列 / 127件の回答」）
  - **列一覧テーブル**:
    | 列名 | 列タイプ | 集計対象 | サンプル値 |
    |------|---------|---------|----------|
    | タイムスタンプ | 日時 | ☐ OFF | 2026/04/20 14:23:01 |
    | メールアドレス | テキスト | ☐ OFF | user@example.com |
    | 年代 | 単一選択 ▼ | ☑ ON | 30代 |
    | きっかけ | 複数選択 ▼ | ☑ ON | 新聞, テレビ, ラジオ |
    | 自由記述 | 自由記述 ▼ | ☑ ON | とても良かったです |
  - **列タイプ**はドロップダウンで変更可能:
    - 単一選択 / 複数選択 / 数値 / 自由記述 / 集計対象外
  - **集計対象**はチェックボックスでON/OFF
  - **アクション**: 「集計画面へ進む」ボタン
- **操作と遷移**:
  - 列タイプドロップダウン変更 → 即時反映
  - 集計対象チェック → 即時反映
  - 「集計画面へ進む」→ 画面3へ
  - 「CSVを差し替える」→ 画面1に戻る（状態リセット）

### 画面3: 集計画面

- **URL**: `/`（状態遷移）
- **アクセス権限**: 誰でも
- **主な要素**:
  - **ヘッダー**:
    - 「集計結果」
    - 「← 列設定に戻る」リンク
    - 「CSVを差し替える」リンク
  - **タブ**: `単純集計` / `クロス集計`

#### タブ1: 単純集計

- **プレビュー表示**:
  - 各設問の集計結果を上から順に表示
  - 例:
    ```
    Q: あなたの年代は？（n=120）
    20代  45人  37.5%
    30代  38人  31.7%
    40代  25人  20.8%
    50代  12人  10.0%

    Q: このイベントを知ったきっかけは？（複数回答、回答総数=185）
    ※ 分母は「回答総数=185」（選択件数の合計）。各選択肢の割合の合計は100%になる
    新聞   40件  21.6%
    テレビ 55件  29.7%
    SNS    65件  35.1%
    ...

    Q: 自由記述（別シート参照 / 全52件）
    ```
- **アクション**:
  - 「Excel出力」ボタン → 単純集計.xlsxをダウンロード

#### タブ2: クロス集計

- **条件設定エリア**:
  ```
  【フィルタ条件】
    [＋ 条件を追加]

    条件1: [設問 ▼] = [選択肢 ▼]  [×削除]
    条件2: [設問 ▼] = [選択肢 ▼]  [×削除]
    ※ 条件は AND で結合されます

  【集計対象設問】
    [設問 ▼]
  ```
  - 「設問」ドロップダウンには、集計対象ONの全設問が並ぶ
  - 「選択肢」ドロップダウンは、設問を選ぶと連動して更新
    - 単一選択列: その列の全ユニーク値
    - 複数選択列: 分解後の全ユニーク選択肢（「含む」でフィルタ）
  - 条件はいくつでも追加可能（AND結合）
  - [×削除]ボタンで個別削除
- **結果表示エリア**:
  - 該当者数表示（例: 「該当者: 24人 / 全120人中」）
  - 0人の場合は「該当者なし」と表示
  - 集計対象設問の結果を単純集計と同じ形式で表示
- **アクション**:
  - 「Excel出力」ボタン → クロス集計結果.xlsxをダウンロード
    - Excelには「フィルタ条件の明細」「該当者数」「集計結果」を含む

## Excel出力フォーマット

### 単純集計Excel

- **ファイル名**: `単純集計_YYYY-MM-DD.xlsx`（例: `単純集計_2026-04-22.xlsx`）
- **シート構成**:
  - **シート1「集計結果」**: 全設問の単純集計を縦に並べる
    - 設問タイトル（元の列名）、回答数（n=）、選択肢・度数・割合の表
  - **シート2以降「自由記述_Q○」**: 自由記述ごとに1シート
    - シート名は設問コード（Q1、Q2等）または短縮した列名

### 単純集計シート レイアウト例

```
| A                          | B      | C      |
|----------------------------|--------|--------|
| Q: あなたの年代は？         |        |        |
| n=120                     |        |        |
| 選択肢                     | 回答数  | 割合   |
| 20代                       | 45     | 37.5%  |
| 30代                       | 38     | 31.7%  |
| ...                        |        |        |
|                            |        |        |
| Q: このイベントを知ったきっかけは？（複数回答） | | |
| 回答総数=185              |        |        |
| 選択肢                     | 回答数  | 割合   |
| 新聞                       | 40     | 21.6%  |
| ...                        |        |        |
```

### クロス集計Excel

- **ファイル名**: `クロス集計_YYYY-MM-DD_HHMMSS.xlsx`
- **シート構成**:
  - **シート1「クロス集計結果」**:
    - 上部: フィルタ条件一覧（設問名 = 選択肢、AND結合）
    - 中部: 該当者数（例: 24人 / 全120人中）
    - 下部: 集計対象設問の結果テーブル

## UIデザイン方針

- **デザイントーン**: シンプル、業務ツール系（派手にしない）
- **カラー**: ベースは白、アクセント色を1色だけ使う
- **フォント**: システムデフォルト（遊ゴシック / Noto Sans JP等）
- **余白**: 十分な余白を取り、情報量が多くても圧迫感を与えない
- **アクセシビリティ**: 最小限考慮（キーボード操作可能、コントラスト比確保）

## エラーハンドリング

- **CSVパース失敗**: 「このファイルは読み込めません。Googleフォームからエクスポートした.csv（UTF-8）を投入してください。」
- **列が0件**: 「列が検出されませんでした。ファイルが空の可能性があります。」
- **ブラウザ非対応**（例: File APIが使えない）: 「Chrome最新版でのご利用をお願いします」
- **集計中エラー**: 「集計中にエラーが発生しました。ページを再読み込みしてお試しください」

## 操作上の留意事項

- CSVはブラウザを閉じた時点でメモリから消える（意図的にローカルストレージに保存しない）
- ブラウザの戻る/進むボタンでは状態が戻らない（SPAのため）
- 同時に複数のCSVを扱うことはできない
